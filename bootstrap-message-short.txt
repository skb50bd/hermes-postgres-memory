5-command TL;DR for installing the postgres memory provider
==========================================================

If you have a superuser Postgres account and a free Kimi key, this is
the whole install. For the full walkthrough with failure modes and
per-dim configuration, see bootstrap-message.txt.

```bash
# 1. clone the repo
git clone https://github.com/skb50bd/hermes-postgres-memory.git /tmp/hpm

# 2. create the database + role + extensions (asks for superuser pw and new role pw)
PGPASSWORD=*** psql -h <host> -U postgres -d postgres \
  -v dbname=hermes -v rolename=hermes -v pw=*** \
  -f /tmp/hpm/plugins/memory/postgres/sql/000_create_database_and_role.sql

# 3. create the plugin schema (agent_memory + settings + models)
PGPASSWORD=*** psql -h <host> -U hermes -d hermes \
  -f /tmp/hpm/plugins/memory/postgres/sql/000_schema.sql

# 4. install the plugin + skill
HERMES_HOME=~/.hermes/hermes-agent /tmp/hpm/install.sh

# 5. set the provider in config + add the embedder key, then restart
echo "KIMI_API_KEY=***" >> ~/.hermes/.env
hermes config set memory.provider postgres
hermes gateway restart
```

Verify:

```bash
hermes postgres-memory preflight
hermes postgres-memory status
```

If the preflight reports failures, run the diagnose script for a
point-by-point diagnosis:

```bash
/tmp/hpm/plugins/memory/postgres/scripts/diagnose.sh
```
